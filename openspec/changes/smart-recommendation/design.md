## Context

기존 `meal-recommender` 앱의 추천 엔진은 요일 패턴과 최근 기록만 기반으로 합니다. 여기에 실시간 날씨 데이터를 추가하여 상황에 맞는 추천을 제공하고자 합니다. 날씨 API는 무료 Open-Meteo를 사용하며 브라우저의 Geolocation API로 위치를 얻습니다.

## Goals / Non-Goals

**Goals:**
- 현재 위치 기반 날씨(기온, 날씨 코드) 조회
- 날씨 조건에 따른 메뉴 필터링 (비→따뜻한 메뉴, 더위→시원한 메뉴, 추위→뜨끈한 메뉴)
- 기존 패턴 기반 추천과 날씨 기반 추천을 통합하여 최종 추천 결정
- 홈 화면에 현재 날씨 정보 + 시간대 아이콘 표시

**Non-Goals:**
- 시간별/주간 예보
- 미세먼지 등 대기질 정보
- 날씨 기반 알림/푸시
- GPS 권한 거부 시 대체 지역 설정 (기본: 서울)

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| 날씨 API | Open-Meteo (open-meteo.com) | 무료, API 키 불필요, 한국 날씨 커버 |
| 위치 정보 | navigator.geolocation | 브라우저 표준 API, 간단함 |
| 날씨 조건 분류 | WMO Weather Code | 비(51-67, 80-82), 눈(71-77, 85-86), 흐림(2-3), 맑음(0-1) |
| 온도 기준 | 28℃↑ 더움, 5℃↓ 추움, 그 사이 보통 | 한국 기후에 맞는 직관적 기준 |
| 추천 우선순위 | 날씨 기반 추천 → 요일 패턴 → 최근 기록 → 기본 제안 | 날씨가 가장 실시간 영향력 높음 |
| 날씨 없을 때 | 기존 추천 그대로 유지 | 위치 권한 거부/API 실패에도 앱 동작 보장 |

## Risks / Trade-offs

| Risk | Mitigation |
|------|-----------|
| 사용자가 GPS 권한 거부 | 기본 위치(서울 좌표)로 fallback, 추천은 계속 동작 |
| Open-Meteo API 장애/지연 | 5초 타임아웃, 실패 시 날씨 없는 기존 추천으로 폴백 |
| 날씨만 보여주고 패턴 무시 | 날씨 추천 점수 + 패턴 추천 점수를 가중치 합산하여 결정 |
| GPS 정확도 낮음 (실내) | 도시 단위 날씨만 필요하므로 정밀도 불필요 — rough position 허용 |
