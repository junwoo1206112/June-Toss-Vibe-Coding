# 뭐먹지? 개인정보 처리방침

시행일: 2026년 6월 15일

뭐먹지?는 사용자의 개인정보를 중요하게 생각하며, 서비스 제공에 필요한 최소한의 정보만 처리합니다.

## 1. 처리하는 정보

- 게스트 식사 기록: 음식 이름, 식사 구분, 즐겨찾기 여부, 기록 시각을 추천과 통계 제공을 위해 사용자의 기기에 저장합니다.
- 토스 로그인 식별자(`userKey`): 사업자 인증 후 계정 동기화 기능을 제공하는 경우 사용자별 서버 기록을 구분하기 위해 처리합니다.
- 위치 정보: 현재 날씨 조회를 위해 기기에서 일시적으로 사용합니다. 위치 좌표는 뭐먹지? 서버에 저장하지 않습니다. 위치 제공을 거부하면 서울 시청 인근 좌표를 기준으로 날씨를 조회합니다.
- 이용 이벤트: 추천 노출, 추천 조건 선택, 추천 메뉴 선택, 식사 기록 성공 여부를 서비스 개선을 위해 수집합니다. 이벤트에는 토스 사용자 식별자와 직접 입력한 음식 이름을 포함하지 않습니다.

## 2. 처리 목적

- 게스트 기기 저장과, 제공되는 경우 토스 로그인 및 사용자 식별
- 식사 기록 저장, 조회, 수정 및 삭제
- 시간, 날씨 및 과거 기록을 활용한 메뉴 추천
- 식사 통계 제공
- 추천 흐름과 서비스 이용성 개선

## 3. 보유 및 이용 기간

- 게스트 식사 기록은 사용자의 기기에 저장되며 앱에서 직접 삭제할 때까지 보관합니다.
- 계정 동기화 식사 기록은 사용자가 앱에서 직접 삭제하거나 서비스 이용이 종료될 때까지 서버에 보관합니다.
- 사용자는 설정의 `모든 식사 기록 삭제` 기능으로 저장된 기록을 즉시 삭제할 수 있습니다.
- 브라우저 세션에 저장되는 로그인 토큰과 사용자 식별자는 로그아웃하거나 세션이 종료되면 삭제됩니다.

## 4. 외부 서비스 이용

- 토스: 사업자 인증 후 계정 동기화 기능을 제공하는 경우 로그인과 사용자 인증
- Supabase: 인증 중계 API와 식사 기록 데이터베이스 운영
- Open-Meteo: 현재 위치 또는 서울 기준 날씨 조회
- 앱인토스 Analytics: 익명화된 화면 노출 및 클릭 이벤트 분석

각 서비스는 해당 요청을 처리하는 범위에서 정보를 처리합니다. 뭐먹지?는 서비스 제공 목적 외로 개인정보를 판매하거나 임의 제공하지 않습니다.

## 5. 이용자의 권리

사용자는 앱 설정에서 자신의 식사 기록 전체를 삭제할 수 있으며, 고객문의 이메일로 개인정보 열람, 정정, 삭제 또는 처리 정지를 요청할 수 있습니다.

- 식사 데이터는 토스 로그인으로 확인한 사용자 식별자를 기준으로 분리합니다.
- 브라우저가 임의로 전달한 사용자 식별자를 신뢰하지 않고 서버에서 토스 로그인 정보를 다시 검증합니다.
